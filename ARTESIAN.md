# ARTESIAN

Artesian is a multi-agent context orchestration system. It is designed to be useful in the smallest mode first: memory tools that work with existing agents without changing normal workflows.

## Names

- Mímisbrunnr: memory.
- Hvergelmir: optional sandbox runtime.
- Bifröst: TUI.
- Yggdrasil: layered context tree.
- Erindi: task.
- Þing: queue.
- Galdr: committed task.
- Óðinn / master, Þórr / worker, Týr / judge: orchestration roles.

## Principles

- MCP-first integration.
- Pluggable agent adapters.
- Pluggable memory backends.
- Non-intrusive modes.
- Files-first bootstrap, vector-ready architecture.
- Deterministic memory drill-down via `node_id`.
- No private infrastructure assumptions.
